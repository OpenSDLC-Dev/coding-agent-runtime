Coding Agent Runtime

在容器里跑 Claude Code，对外暴露一套可对话、可观测、有标准规范的 HTTP 服务；外加一个连容器、看 spec、对话、看 trace 的测试界面。

01

背景与目标

把 Claude Code 跑在容器里对外服务：调用方与容器内 agent 单/多轮对话完成编码任务；agent 能读写挂载的工作区、执行白名单内 bash；服务暴露 OTel 遥测与 OpenAPI 规范，另配测试界面。

目标 · In scope

  • SDK 调自带内置 CLI（不单独装二进制），基于 SDK 提供 HTTP API（会话增删查 + 流式对话）
  • OpenAPI 3.1 + Swagger UI，CORS 开放
  • OTel：metrics / logs / traces（OTLP），看「每轮→工具→LLM」链路与 token/费用，每轮回传 traceId
  • Bash 命令白名单
  • 容器无状态：CLAUDE_CONFIG_DIR 与工作目录动态挂载，可丢弃/池化
  • 模型后端可配（ANTHROPIC_BASE_URL/KEY/模型名），支持第三方兼容端点
  • 测试界面：连接 / 看 OpenAPI / 对话 / 看 trace

非目标 · Out of scope（本期）

  • 多租户鉴权（既定：无 HTTP 鉴权 · 可信网络）
  • 多容器编排 / 暖池 / 一会话一容器控制面（第②层，后期）
  • 跨主机 SessionStore（挂载目录即可）
  • 计费级成本核算（total_cost_usd 仅客户端估算）
  • 测试界面是开发/验证工具，非终端用户产品
02

总体架构

独立测试前端经 HTTP(CORS) 调用本地 Docker 上的 runtime 容器；容器内 SDK 驱动 CLI，每轮 query({ resume }) 从挂载目录读写状态；遥测经 OTLP 进自带后端栈，前端按 traceId 深链 Jaeger。

实线 = 请求/导出路径；虚线橙 = 前端按 traceId 深链 Jaeger。容器本身无状态，状态都在底部挂载目录。

03

配置（环境变量 / .env）

配置走环境变量；query() 经 options.env 透传给 CLI 子进程。

⚠️ TS 关键坑 options.env 是整体替换子进程环境，不是合并——必须 { ...process.env, ... }，否则丢 PATH/HOME 导致 spawn 失败。

04

HTTP API 设计

REST（会话增删查）+ SSE（流式），单份 OpenAPI 3.1，Zod 为唯一 schema 源。

端点
GET    /healthz                  健康检查
GET    /config                   { defaultModel, jaegerBaseUrl, version }（供前端）
GET    /openapi.json   /docs     OpenAPI 3.1 + Swagger UI

POST   /sessions                 创建+首轮；text/event-stream 返回本轮事件
GET    /sessions                 列出（id/状态/模型/轮次/累计 token）
GET    /sessions/:id             会话信息
GET    /sessions/:id/transcript  完整 transcript
POST   /sessions/:id/turns       追加一轮（resume）；SSE 返回
POST   /sessions/:id/stop        中止当前轮（abort）
DELETE /sessions/:id             删除会话

SSE 事件（名取自 SDKMessage.type）

event载荷
initsessionId, traceId, model, cwd, tools
assistant文本块 + tool_use 块
tool_result工具输出
token逐 token 增量（开 includePartial 时）
resultsessionId, traceId, usage, cost, num_turns
error / aborted服务级错误 / 中止
SSE 经 POST 对话端点是 POST（带 prompt body），浏览器 EventSource 只支持 GET——前端用 fetch + ReadableStream 读 text/event-stream。
OpenAPI 描述 SSE 用 text/event-stream + 单事件 schema（3.1 表达不了"连续流"）；事件判别用 const；可选补丁成 3.2 itemSchema。
05

会话执行模型 · 无状态每轮 resume

每个 HTTP 轮次 = 一次独立 query()；多轮靠挂载的 transcript，容器因此真正无状态、可池化。

流程

  • 首轮：query({ prompt, options:{ cwd:'/workspace', model, env, hooks, systemPrompt:{preset:'claude_code'}, settingSources:['user','project'] } })，从 system/init 捕获 session_id 返回
  • 后续轮：附 resume: sessionId，CLI 从挂载 CLAUDE_CONFIG_DIR 读历史续写
  • 不持有长连接 generator；要中途插话再引入流式输入模式
⚠️ 铁律 cwd 容器内固定 /workspace。transcript 目录由 cwd 路径编码而来，路径一变 resume 静默失败、历史丢失。
取舍 代价是不能在 agent 跑一半时插话——可 /stop 中止后再开下一轮。换来极简、可池化、与"无状态+挂载"完全契合。
06

权限与安全

容器是真正隔离边界。CLI 以 bypassPermissions 无人值守运行，安全靠两层强制 + 容器边界。

两层强制

  • PreToolUse hook（matcher:'Bash'）：拆 && || ; |、剥 timeout/nice 包装器，任一子命令不在白名单即 deny。hook 跑最前、连 bypass 都拦、覆盖子 agent
  • disallowedTools 兜底：Bash(curl *) / Bash(wget *) / Bash(sudo *) / 危险删除（deny 永远赢）
  • 白名单（含编码工具）：git/gh/node/npm/npx/python/python3/uv/uvx + 常用只读/文件命令。注意 gh/git 能联网 = 有意出网口，与 deny curl/wget 相反

边界与密钥

  • allowedTools 仅"自动批准"，不当限制
  • settingSources:['user','project']：user=entrypoint 写入 $CLAUDE_CONFIG_DIR/CLAUDE.md 的固定 guidelines，project=挂载仓库自带 ./CLAUDE.md（合并）。⚠️ 开 'project' 同时载仓库 .claude/settings.json = 信任扩到被执行代码；安全仍靠 SDK hooks+disallowedTools 强制
  • 文件限定挂载的 /workspace；容器层加出网 egress 白名单（P3）
  • 密钥仅经 env/.env，不入镜像/日志/trace
07

可观测性 / OTel

CLI 原生 trace 是 beta，且 SDK（stream-json）路径有缺口——必须自建一层 span 才能看到完整调用树。

关键发现 · issue #53954（已"不修"） SDK 用 --input-format stream-json 驱动 CLI，原生只产 claude_code.llm_request span，丢了 interaction(每轮) 与 tool 层。所以「完整调用链」需要我们自己补。

自建 span 方案

  • Node 服务自建 agent.session → agent.turn span
  • turn span 上下文序列化为 TRACEPARENT 注入 options.env，CLI 原生 span 挂其下，形成统一 trace
  • 从消息流按 tool_use_id 补 tool span（补回工具层）
  • 从 SDKResultMessage 读 usage/total_cost_usd 打到 span（message.id 去重）
  • 跨轮用 gen_ai.conversation.id = sessionId 在后端归并
claude_code.interaction claude_code.llm_request claude_code.tool
后端栈（自带，可换） docker-compose：OTel Collector → Jaeger(:16686，深链目标) + Prometheus。Collector 可加 transform 把 claude_code.* 映射成 gen_ai.*。
注意点 trace 需 beta flag；禁 console exporter（污染 SDK stdout）；一次性容器降导出间隔防丢；MiniMax 成本估算失真、仅 token 可信、看上游需 PROPAGATE_TRACEPARENT。
08

容器与挂载

镜像无状态、可复现；状态全在运行时挂载进来的 /workspace 与 /claude-config。

Dockerfile（要点）+ docker run（本地）
FROM node:22-bookworm-slim          # 非 alpine：glibc，保 optional 原生依赖
RUN apt-get install -y git gh ripgrep curl ca-certificates libstdc++6
# 装 uv，并由 uv 提供 python 3.12（Debian 主源无 3.12）
RUN curl -LsSf https://astral.sh/uv/install.sh | sh && uv python install 3.12 --default
# CLI 运行时随 SDK 自带，pnpm install 即带；不单独装二进制
RUN pnpm install # 不裁 optional 依赖；非 root 用户；预建并 chown 挂载点

# 运行（state 都在挂载里）
docker run -p 8080:8080 \
  -v $PWD/work:/workspace  -v $PWD/cfg:/claude-config \
  -e CLAUDE_CONFIG_DIR=/claude-config  --env-file .env  runtime:latest
三个高频坑 ① CLI 随 SDK 自带（默认 pathToClaudeCodeExecutable），不装二进制；版本随 SDK → 靠 pin SDK 来 pin CLI。② entrypoint 初始化：写 $CLAUDE_CONFIG_DIR/CLAUDE.md（固定 guidelines）+ 跑 gh auth setup-git（git 也用 GH_TOKEN）；bind mount 的 UID/GID 必须对齐容器用户。③ 镜像勿裁 optional 依赖、用 slim Debian 非 alpine。

分阶段实施计划

每阶段以"可运行 + 可验证"为准（CLAUDE.md §4 目标驱动）。

P0 · 骨架

骨架

pnpm workspace + Biome/vitest；runtime：Hono + healthz/config + Dockerfile（SDK 自带 CLI + git/gh/uv/py3.12）+ entrypoint（写 $CLAUDE_CONFIG_DIR/CLAUDE.md）+ 单轮 runTurn（systemPrompt preset + settingSources:['user','project']）+ SSE 出流。

✓ 验收：本地 docker run 后 POST /sessions 能流式返回 agent 输出；容器内 agent 遵循固定 guidelines
P1 · 多轮+界面

多轮 + 会话 + 界面骨架

每轮 resume、Session Registry、/sessions/* 全套、OpenAPI+Swagger UI+CORS、模型后端可配（MiniMax 跑通）；web：连接+spec+SSE 对话。

✓ 验收：多轮上下文连续；MiniMax-M3 完成真实任务；界面能连/看 spec/对话
P2 · 可观测

可观测性 + 界面 trace

telemetry.ts、session/turn span、TRACEPARENT 注入、工具树、usage→span、compose 后端栈；透出 traceId；界面深链/内嵌 Jaeger。

✓ 验收：Jaeger 看到「turn→tool→llm_request」链路与 token；界面一键看 trace
P3 · 安全韧性

安全 + 韧性

PreToolUse Bash 白名单 + disallowedTools + egress 白名单 + 容器硬化；Last-Event-ID 重放/心跳；abort 接线。

✓ 验收：越权 bash 被拦并回有意义信息；断线可重连；中止生效
P4 · 可选

规模化（可选）

第②层编排：控制面 spawn 一会话一容器、动态挂载、生命周期/暖池/路由。

✓ 验收：多会话并发各自隔离；暖池冷启动秒级

开放问题 / 待验证

MiniMax 兼容性
/anthropic 端点对 streaming、usage 字段、prompt caching、enhanced-telemetry beta 的支持；辅助"小模型"的确切环境变量名（随 CLI 版本变）。
issue #53954 是否已悄悄修复
streaming 模式缺 interaction/tool span——对 pin 的 CLI 版本实测，可能已修但状态未改。
单 string prompt 是否恢复完整 span 树
缺口指向 --input-format stream-json；单 string prompt 可能走不同路径——实测确认。
TRACEPARENT 采纳细节
是否需同带 TRACESTATE 与 sampled 标志才被 CLI 采纳——隔离环境验证（不能在 SDK 主通道用 console exporter）。
自带 CLI headless 稳定性
SDK 内置 CLI 在容器无头下被 query() 驱动（含 stream-json、resume）是否稳定——P0 实测；必要时回退显式装 CLI + pathToClaudeCodeExecutable。
会话删除清理策略
删除时挂载状态保留/归档/删除，与 30 天自动清理的取舍。
